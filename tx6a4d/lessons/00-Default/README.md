函数介绍
===

这里出现两种`输入`量： **Button（按键）** 和 **Stick（旋钮）** 。

一个正常的“十字”摇杆其实就包含了“水平”方向和“垂直”方向的两条旋钮。

*Tx6a4d* 遥控器，就包含了肩部 4 个 **Button（按键）** 和 6 个 **Stick（旋钮）** （两个十字摇杆各包含两个，小记 4 个，再加上2个常规旋钮）。

**Button（按键）** 只有两种状态， `true`（1） 或 `false`（0），对应`“被按下”`和`“未按下”`，通过传入 **index（序号）** 参数，可以通过 `getButton(index)` 函数获取相应按键的状态。

**Stick（旋钮）** 的状态是个模拟量，它的读数，即`getStick(index)`函数的返回值，范围是 `[-127, -126, ..., -1, 0, 1, ..., 126, 127]`（包含 -127 和 127），为 0 则说明当前在行程的中点，绝对值越大，则说明其偏离中点越远。其符号（正负号）则对应它偏离中点的方向。

**index（序号）** 为对应输入量的编号。

因为 **Button（按键）** 总共有4个，所以 getButton 可传入的 **index（序号）** 的值有 `[0, 1, 2, 3]` 四种。

因为 Stick (旋钮)总共有6个，所以 getStick 可传入的 **index（序号）** 的值有 `[0, 1, 2, 3, 4, 5]` 六种情况。

> 这里编程所使用的是标准 C 语言代码，所以数组下标的计数由 0 开始。

具体按键编号与实际按键的对应关系，请查看电路板上的标识或所附说明文档

这里又出现了一种 `输出` 量： **Channel（通道）**，也叫信道（信号的通道）。

**Channel（通道）** 是相互独立的数据传输管道。

*Tx8a4d* 上包含 16 个逻辑通道，每个通道都是一个`字节`（8位），每个通道的设置范围也是 `[-127, -126, ..., -1, 0, 1, ..., 126, 127]` 。

不管输入如何变化，遥控器每次对外发送的数据，就是这 16 个通道（字节）的数据。

接收机（被遥控的设备），每次接收到的数据，也是这 16 个通道（字节）的数据，它会根据这16个字节的数据控制相关的输出设备（舵机、马达、LED 等等）。

所以，自定义遥控器的功能，其实就是建立`输入`量（按键/旋钮）与`输出`量（通道）之间的对应关系。

其中，`setChannel(index, value)`函数的功能，就是将 **index（序号）** 的 **Channel（通道）** 设置为 `value` 。
**index（序号）** 取值范围是 `[0, 1, 2, 3 , ..., 15]`，也是合计 16 个。

> 但接收机如何解释这些数据是由接收机的程序决定

因为发射器需要不断给接收机发送数据，而且这些数据体现了输入量的变化，所以主控芯片会重复计算各通道的数值并发送出去。

这里的“重复计算”就发生在 `app()` 函数当中，可以由用户自行定义，以实现自由的组合搭配。

系统内部调用 `app()` 函数的频率是 `50Hz`，也就是每 1 秒钟发送 50 次数据包，与常规模拟舵机的输出频率相同。

当然我们在重新设置 **Channel（通道）** 数值的时候，也可能会用到它之前（上一轮执行）的数值，可以通过 `getChannel(index)` 函数来获取。但如果已经调用过 `setChannel(index, value)` 的话，之前的数值就被覆盖为新数值，再调用`getChannel(index)` 只能获取新值了。
